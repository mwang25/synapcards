Design Notes

Sections

Optional featured cards, latest cards added, pinned cards which are for the author for his own personal favorite cards.

Cards

  1. Top row of all cards should have card id on the left, and icons in the middle, and a drop down menu on the right. Card is is a hyperlink to the card itself.
  2. Heart icon for likes. Red color is like by the author/owner of the card, pink color is like by other people but not the author. Below the icon is the total number of likes. Or maybe the heart is red if there is any likes and the counter says liked by you and x others.
  3. Eye icon for number of views. Too fine grained?
  4. If this card is not owned by the viewer, then a bell icon (or telescope) for getting notified of updates to this card.

Cards can be private, public, or view permissions granted to specific users. Should we ever allow other people to write to someone else's card? Seems hard to deal with. They could just create a shared account.

Card Drop down menu

  1. If this card is owned by the viewer, then edit option.
  2. If this card is owned by the viewer, then sharing option. Sharing allows the owner to make the card private, share with other synapcard users, or make the card public.
  3. If this card is owned by the viewer, then delete option.
  4. If this card is not owned by the viwer, then copy option. Copy will create a new card owned by the current viewer. Once the copy is made, the copy is independent of the original and can diverge from the original card.
  5. Everyone will have a meta info option, which shows meta info about the card, such as creation date, last update date, etc.

Login

Support for users logging into their own accounts using oath/google identification. Login information is needed to implement secure email updates, sharing, and support for multiple users.

Each user can optionally create a profile page for him/herself. Profile page can also show stats about the account, e.g. how many cards the user has created, how many added last month, distribution of tags, total views(?), total likes(?), when the user joined synapcards, etc. Need at least one setting: timezone the user lives in.

Privacy

Every card, section, page will have privacy control. They can be set to private, shared with specific synapcard users, or public.

Releases

First release (7/21/17): static site. DNS handled by GoDaddy, but redirected to Google storage buckets. Static pages in the storage buckets. Made a logo. Google analytics tag in all pages. Google ad tag in all pages, but Google rejected my site because it was not finished yet.

Second release (9/21/17): Dynamic site. DNS transfered Google. Bought SSL cert from SSLMate, so my site can be accessed via https. Site served by Google AppEngine running python webapp2 and Jinja templates. Cards stored and served from DataStore, a noSQL database. Simple user profile page. Very hacky featured cards page with ability to see previous featured cards.
